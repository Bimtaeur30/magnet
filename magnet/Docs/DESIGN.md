## 2. 게임 개요

| 항목 | 내용 |
|------|------|
| 가제 | **MAGNET SQUARE** |
| 장르 | 하이퍼 / 하이브리드 캐주얼 블록 퍼즐 |
| 한 줄 소개 | 자석 축에 블록이 자동 흡착되고, 정사각형을 완성하면 폭발하며, 매 턴마다 보드가 90° 회전하는 블록 퍼즐 |
| 레퍼런스 | Block Blast! (블록 퍼즐 골격), 무한의 계단 (점수 기반 스킨 해금) |
| 플랫폼 | 모바일(iOS/Android) 우선, **세로 화면·단일 핸드** |
| 시점 | **사이드뷰 2D** (격자 보드 정면) |
| 세션 길이 | 1~3분 |
| 팀 | 개발자 4명 (기획·아트 별도 인력 없음) |
| 프로토타입 목표 | **거의 전체 기능** — 코어 루프 + 점수 + 스킨 해금/인벤토리/장착/저장까지 |

### 2.1 핵심 차별점

1. **자석 자동 흡착** — x축 위치만 정하면 축 방향으로 이동해 **블록·자석**에 먼저 닿는 지점에 스냅. 경로에서 아무것도(블록·자석) 못 만나 경계만 닿으면 **그 위치에는 놓을 수 없음**.
2. **매 턴 90° 회전** — 자석 축 중심으로 보드 전체(부착 블록 포함) 회전. 공간 추론이 매 턴 달라짐.
3. **정사각형 테두리 클리어** — 자석 축 중심 N×N(홀수) **테두리** 완성 시 폭발. **테두리만 파괴**, **바깥 칸은 재배치(달팽이)**, **내부** 유지. 부착 후 보드는 **칸 단위**.

### 2.2 심사·방향성 메모

- 블록 퍼즐 장르 CPI 특성을 고려해, **접근성 높은 조작·즉각 재도전**을 우선한다.
- **스킨 해금**은 “다음 스킨까지 한 판 더” 리텐션 장치로 운용한다 (게임플레이 영향 없음).

## 3. 핵심 게임플레이 루프

한 턴(블록 1개 배치)의 **고정 순서** — 이 순서가 게임의 핵심 규칙이다.

```
블록 선택 → x축 배치 → 자석 흡착·부착 (이후 칸 단위)
  → 경계 이탈 판정 (게임오버)
  → 최내곽 정사각형 폭발 → 바깥 칸 재배치 → 재검사 (연쇄)
  → 자석 축 90° 회전 (보드·칸 동반)
  → 다음 블록 후보 갱신
```

| 단계 | 설명 |
|------|------|
| 1. 블록 후보 | 화면 하단에 **3개** 무작위 블록 제시. 하나 선택 |
| 2. x축 배치 | 터치·드래그로 **좌우(x)만** 조정. 손을 떼면 흡착 시작 |
| 3. 자석 흡착 | 자석 축 방향으로 이동 → 경로상 **가장 먼저** 닿는 **기존 칸 또는 자석 축** 직전에서 스냅 부착. **경계만** 만나고 칸·자석을 못 만나면 **배치 실패(놓을 수 없음)** — 경계에 부착하지 않음. 부착 완료 시 **칸 단위로 분해** |
| 4. 경계 이탈 | 부착 **성공** 직후, 고정 경계 밖으로 나간 칸이 있으면 **즉시 게임오버** |
| 5. 폭발·재조립 | 회전 **전** 좌표. 완성된 N 중 **가장 안쪽만** 테두리 파괴 → 바깥 칸 부채꼴 목표 재배치 → 다시 판정. 없을 때까지 반복. 연출 중 입력 잠금. 점수 = 파괴 테두리 칸 |
| 6. 90° 회전 | 연쇄 재조립 **종료 후**, 자석 축 기준 **시계방향 90°** (방향은 SO로 설정 가능). 모든 부착 칸 동반 회전 |
| 7. 반복 | 후보 1개 소모 시 새 블록으로 채움. 3개 모두 놓을 수 없으면 게임오버 |

> **주의:** `판정·재조립 연쇄 → 회전` 순서는 절대 바뀌지 않는다. 연쇄 중간에 회전하지 않는다.

## 4. 핵심 시스템

### 4.1 보드와 자석 축

- **N×N 정사각 격자** (프로토타입 기본값: `9`, ScriptableObject로 변경 가능).
- **자석 축**: 보드 정중앙 1칸에 고정. 이동하지 않음. 폭발·회전의 기준점.
- 자석 축 자체는 블록이 아님.

### 4.2 블록 공급

- 항상 **3개 후보** 동시 제시.
- 형태 풀 예시: `1×1`, `1×2`/`2×1`, `1×3`/`3×1`, `2×2`, `L자(3~4칸)` 등.
- 1개 사용 시 해당 슬롯만 새 블록으로 교체.
- **게임오버**: 3개 모두 어떤 위치에도 둘 수 없을 때.

| 형태 | 칸 수 | 난이도 | 비고 |
|------|-------|--------|------|
| 1×1 | 1 | 낮음 | 구원용 |
| 1×2 / 2×1 | 2 | 낮음 | 기본 라인 |
| 1×3 / 3×1 | 3 | 보통 | 테두리 변 |
| 2×2 | 4 | 보통~높음 | 내부 채움 |
| L자 | 3~4 | 높음 | 변수 창출 |

출현 가중치·난이도 곡선 수치는 플레이테스트 후 확정.

### 4.3 배치·자석 흡착

- 플레이어는 **x축 위치만** 결정. y(축 방향) 정착은 흡착 규칙이 결정.
- 흡착 경로에서 **기존 블록 또는 자석 축 `(0,0)`** 에 닿으면 그 **직전** 칸에 스냅 부착.
- 경로에서 블록·자석을 **하나도 못 만나** 보드 **Y 경계**에만 도달하면 → **그 x에서는 놓을 수 없음** (부착하지 않음). 빈 보드에서도 자석에 닿지 않는 열은 배치 실패.
- 부착 후 플레이어가 직접 이동 불가. 위치 변경은 **회전·클리어 재배치**로만 발생.
- 부착 완료 시점부터 Domain·View 모두 **칸 단위** (`OccupiedCell`). 후보 슬롯의 멀티 쉐이프는 부착 전까지만 블록 단위.
- **프리뷰(가이드 라인)** 권장 — 스냅 위치·배치 불가를 미리 표시.

### 4.4 회전

- 블록 **1개 부착마다** 1회, **클리어 재조립 연쇄 종료 이후** 90° 회전.
- 회전으로 인한 경계 이탈은 별도 게임오버로 처리하지 않음 (부착 시점에 이미 걸러짐).

### 4.5 폭발(클리어) 판정·재조립

- 자석 `(0,0)` 중심 **홀수 N×N** (3×3, 5×5, 7×7 …). 한 변 길이 = N, 반경 `half = (N−1)/2`.
- **발동 조건:** 해당 N×N **테두리** 칸(체비셰프 거리 = `half`)이 끊김 없이 칸으로 채워짐. 자석 칸 `(0,0)`은 판정·점유 대상 아님.
- **다중 완성:** 여러 N이 동시에 만족해도 **가장 안쪽(최소 N)만** 터뜨린다. 재조립 후 다시 판정.
- **파괴 범위** (한 웨이브):
  - N×N **테두리** 칸 — **제거(점수)**
  - N×N **내부** (`max(|x|,|y|) < half`) — **유지**
  - N×N **바깥** (`max(|x|,|y|) > half`) — **삭제하지 않음**. 재배치 대상
- **보드 유닛:** 부착 이후 **칸(`OccupiedCell`)만**. 멀티칸 블록 연결 개념 없음.
- **재배치 (Domain이 진실):**
  - 순서: `half+1` 링부터 바깥으로, 링 안은 12시→시계방향
  - 목표: 자석 방향 **90° 부채꼴** ∩ 서클(반지름=자석 거리) 안 미예약 빈칸 중 **자석에 가장 가까운 칸**
  - 배정 즉시 출발 칸 비움·목표 예약. 테두리 메우기가 목표가 아님(같은 면 밀도 유지가 자연 결과)
  - Domain이 웨이브 단위로 파괴+이동 선확정 → 연쇄 끝까지 시뮬레이션
- **연출 (Presentation):** LitMotion. 튕김(기본 3칸) → 고정 반지름 **시계 1바퀴** → 착지. 비행 중 자전(공전 무관), 착지 시 보드 격자 90° 스냅. 스태거로 링이 겹쳐 흐름. 연출 중 입력 잠금.
- **내부 완성(풀 클리어):** 테두리+내부까지 채워도 파괴는 테두리만. `isFullClear` 플래그는 점수용(SCRUM-23).
- **판정 시점:** 부착 직후 및 각 재조립 웨이브 후, **회전 전** 좌표.

### 4.6 게임오버 조건

1. 부착 **성공** 직후, 어떤 칸이라도 보드 경계 밖 (형태가 커서 자석·블록에 붙은 뒤에도 삐져나온 경우).
2. 3개 후보가 **어떤 x에서도** 블록·자석에 흡착될 수 없을 때 (전부 `NoSnapTarget` 등과 동치의 배치 불가).

> **구분:** “경계만 만나 그 자리에 못 놓음”은 **그 시도의 배치 실패**이지, 그 자체로 즉시 게임오버는 아님. 후보 3개 전부가 어떤 x에도 못 붙을 때 게임오버(조건 2).

### 4.7 점수

- 폭발 **파괴 테두리 칸** 수에 비례한 기본 점수 (재배치 칸은 점수 없음).
- 한 턴 안 **연쇄 웨이브**의 테두리 파괴 칸을 합산.
- **풀 클리어**(`isFullClear`: 테두리+내부까지 채움) 추가 보너스 (수치 TBD, SCRUM-23).
- 연속 턴 폭발 시 **콤보 체인** 배율 (수치 TBD).
- 게임 종료 시 최종 점수 vs **베스트 스코어** 비교 노출.

### 4.8 스킨 시스템 (코스메틱)

| 기능 | 설명 |
|------|------|
| 해금 | 누적 점수(또는 베스트 스코어)가 **N점** 도달마다 새 스킨 해금 |
| 인벤토리 | 보유·잠금 상태 목록 UI |
| 장착 | 선택 스킨을 인게임 블록 비주얼에 적용 |
| 저장 | 보유 목록 + 현재 장착 스킨 **영구 저장** |
| 영향 | 게임플레이 수치에 **영향 없음** |

### 4.9 추후 확장 (프로토타입 범위 밖)

- 가변 회전 각도/방향 모드, 컬러 보너스, 자석 축 파워업, 일일 시드 경쟁, 광고 부스터·코인 등.

## 6. 기술 아키텍처 (프로젝트 규칙)

### 6.1 스택

| 영역 | 선택 |
|------|------|
| Unity | 6000.3.x, URP 2D |
| DI | **Reflect** |
| Async | **UniTask** (신규 Coroutine 금지) |
| Tween | **LitMotion** (DOTween 금지) |
| 이벤트 | **EventChannelSO** + `GameEvent` 파생 클래스 |
| 데이터 | **ScriptableObject** (보드 크기, 블록 형태, 스킨, 점수 테이블) |

### 6.2 책임 분리 (SOLID)

| 레이어 | 책임 |
|--------|------|
| Data | `BoardConfigSO`, `BlockShapeSO`, `SkinSO`, `ScoreConfigSO` |
| Domain / Rules | 격자 상태, 흡착 시뮬, 폭발 판정, 회전, 게임오버 — **순수 로직 우선** |
| Application | 턴 진행 오케스트레이션, 점수·스킨·세이브 |
| Presentation | 블록·보드 뷰, 이펙트, UI |
| Input | x축 드래그 → 배치 요청 |

- MonoBehaviour는 표현·입력·DI 바인딩. 규칙 판정은 테스트 가능한 클래스로 분리.
- 오브젝트 간 통신은 `EventChannelSO` (직접 C# event / UnityEvent 지양).

### 6.2.1 Reflex vs SerializeField (확정)

| 대상 | 연결 |
|------|------|
| ScriptableObject·**프로젝트 에셋**(프리팹 등) | Inspector `[SerializeField]` — **Reflex 사용 안 함** (프리팹·씬 분리해도 유지) |
| **씬의 다른 GameObject·MonoBehaviour**·런타임 서비스 | Reflex `[Inject]` — **SerializeField 금지** (프리팹화 시 참조 끊김) |

이름만으로 역할이 불명확한 `[SerializeField]`에는 `[Tooltip]`(한국어)을 붙이고, 문구는 [`INSPECTOR_TOOLTIPS.md`](./INSPECTOR_TOOLTIPS.md)에 기록한다.

### 6.2.2 보드 좌표 (확정)

- **격자 좌표:** 자석 = **(0, 0)**. N=9 유효 보드 = `gx, gy ∈ [-4 .. 4]`. 자석 옆 = `(1, 0)`.
- **월드:** `GridToWorld` — `(gx * cellSize, gy * cellSize)`, 자석 = `Vector2.zero`.
- **변환:** `BoardCoordinates.GridToWorld` / `WorldToGrid` (+ `IsInBounds`).
- **점유 상태:** `BoardGrid` — `Dictionary<Vector2Int, bool>` (배열 없음). 보드 밖 칸도 키로 넣은 뒤 `IsInBounds`로 경계 이탈 판정 (Phase 4).
